### Visão Geral do Desafio

O desafio consiste em desenvolver um sistema de agendamento com 3 entidades principais (Cliente, Serviço, Agendamento) e seus devidos relacionamentos. A proposta é conduzir você desde a modelagem conceitual (UML), passando pela construção das camadas da aplicação, até a entrega de uma API RESTful robusta e bem estruturada.
